# رفع خطای «Indexing request rejected» در گوگل

## علت معمول
گوگل برای هر آدرس (مثل `/about`) یک درخواست می‌زند. اگر **همان HTML صفحهٔ اصلی** را ببیند یا آن آدرس **۴۰۴** برگرداند، ممکن است ایندکس را رد کند.

## کارهایی که انجام شده
- فایل **`_redirects`** در `public/` اضافه شده تا روی **Netlify / Cloudflare Pages** همهٔ مسیرها به `index.html` با وضعیت **۲۰۰** برگردند (نه ۴۰۴).
- در **sitemap** آدرس‌ها با **اسلش آخر** (مثل `https://activelegend.ir/about/`) هستند تا با هاستی مثل GitHub Pages سازگار باشند.

## چک‌لیست شما

### ۱. وضعیت ۲۰۰ برای همهٔ آدرس‌ها
در مرورگر یا با ابزار آنلاین این آدرس‌ها را باز کنید و مطمئن شوید **کد وضعیت ۲۰۰** می‌گیرند (نه ۴۰۴):
- `https://activelegend.ir/`
- `https://activelegend.ir/about/`
- `https://activelegend.ir/games/`
- `https://activelegend.ir/contact/`

اگر **۴۰۴** می‌گیرید:
- **Cloudflare Pages:** در پروژه → Settings → Builds & deployments → در بخش SPA یا Fallback URL مقدار `index.html` یا مشابه آن را تنظیم کنید تا برای همهٔ مسیرها همان فایل با ۲۰۰ سرو شود.
- **GitHub Pages:** بعد از `npm run build:prerender`، پوشهٔ `dist` را دیپلوی کنید. داخل `dist` باید پوشه‌هایی مثل `about`، `games` و داخل هرکدام یک `index.html` باشد. آدرس‌هایی که در sitemap هست با **اسلش آخر** باشند (مثلاً `.../about/`).

### ۲. لینک داخلی از صفحهٔ اصلی
گوگل باید از **صفحهٔ اصلی** بتواند به بقیهٔ صفحات برود. در سایت شما Navbar لینک به `/games`، `/about`، `/contact` دارد؛ اگر هاست شما با اسلش آخر بهتر کار می‌کند، می‌توانید لینک‌ها را به `/games/`, `/about/`, `/contact/` عوض کنید (اختیاری).

### ۳. گوگل سرچ کنسول
- **Sitemap:** آدرس sitemap (مثلاً `https://activelegend.ir/sitemap.xml`) را در GSC ثبت کنید و بگذارید گوگل دوباره کراول کند.
- **درخواست ایندکس:** برای هر URL که رد شده، در GSC بروید به **URL Inspection** → همان آدرس را وارد کنید → **Request indexing**. اگر هنوز خطا داد، در همان صفحه بخش **View tested page** را ببینید و ببینید گوگل چه HTML/خطایی می‌بیند (مثلاً ۴۰۴ یا صفحهٔ خالی).

### ۴. اگر هنوز رد می‌شود
اگر با ۲۰۰ و sitemap درست هم، باز «Indexing request rejected» می‌گیرید، احتمالاً گوگل محتوای داخل صفحه را خالی یا تکراری می‌بیند (چون اول فقط یک shell خالی لود می‌شود و محتوا با JS پر می‌شود). در آن صورت گزینه‌ها:
- **Prerender واقعی:** در بیلد، هر مسیر با یک headless browser (مثلاً Puppeteer) رندر شود و خروجی HTML هر صفحه در `dist` ذخیره شود.
- **سرویس Prerender:** استفاده از سرویسی که برای ربات‌ها نسخهٔ رندر شدهٔ هر URL را سرو می‌کند.

اول مطمئن شوید همهٔ آدرس‌های مهم با **۲۰۰** و **همان محتوای SPA** لود می‌شوند و sitemap و درخواست ایندکس را دوباره تست کنید.
